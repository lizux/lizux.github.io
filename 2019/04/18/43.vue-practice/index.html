Vue 技巧

  1. 设置别名，简化导入组件的路径

  2. 因为vue data 初始化时不能嵌套定义（一个对象的属性值指向另外一个值），所以有类似需求用字符串来指定属性名，同时还能解决延迟加载的数据问题

  3. 路由链接尽量通过 name 来设置而非直接对 url 硬编码

  4. 路由链接会提前预解析，导致 vue-router 的警告“missing param for named route”，加 v-if 判断修复

  5. 路由组件通过特殊设置来对参数解耦

  6. 自定义组件复用后会有缓存问题：添加 key 值，避免翻页后缓存造成不渲染的问题

  7. 组件提炼为公共组件后会有缓存问题：通过 router 加 key 避免复用组件不刷新的问题，会有副作用：内部组件不会缓存会重复刷新，改用 watch 方法实现

  8. vue 的 v-for 语句会把内部 this 指向全局变量（一般为 undefined）；而非当前 vm 实例的问题：重新绑定 this

  9. 使用 keepAlive 缓存组件后，假如内部有嵌套路由，则不能同时开启多个子路由组件的 meta.keepAlive 属性，需要对嵌套的 router-view 再次包裹 keepAlive（可以使用 include 指定要缓存的组件）

  10. 使用 keepAlive 缓存组件后，由于组件一直在缓存中，所以其 watch 方法会一直调用，假如观察的是 route 参数，会频繁触发，注意性能优化！

  11. vue router 的 beforeRouteUpdate 事件不会在 keepAlive 包裹组件里面触发，可以使用 beforeRouteEnter 或 watch 解决（ 参考1 参考2）

Element UI 技巧

  1. 使用 tab 组件时要优化性能，避免重复渲染

  2. 表单初始值的问题:
    multiselect、 file 类型必须设置初始值为空数组，否则报错
    checkbox 初始值必须为空数组，否则无法实现多选

  3. 表单静态渲染项也得加 prop 属性，否则校验程序不生效；同时还得设置 rules 属性

  4. 去除表单原生的 required 属性，避免重复校验

  5. upload 组件实现限制大小

    目标：要在选择文件而非提交表单时实现限制大小功能
    过程：因为没有 before-change 钩子函数，所以只能用 on-change 事件，在检测到文件超过限制时，调用 clearFiles() 方法

    注：在 v-for 循环中 upload 组件的 ref 会变成一个数组，而不是单个对象，即使该 ref 值是唯一值，所以需要特别处理 [0] 才能访问

  6. upload 组件实现返回信息在 file list 中显示

  7. input 组件支持回车事件

    Element 的 input 组件没有实现 keyup 事件，所以要监听回车事件必须使用 @keyup.native.enter='' 形式
